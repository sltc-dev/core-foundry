### 命名约定

- **文件命名**：`kebab-case.tsx` 或 `snake_case.py`
- **变量命名**：`camelCase` 或 `snake_case`
- **常量命名**：`UPPER_SNAKE_CASE`
- **组件命名**：`PascalCase`

### Git 工作流

1. **分支命名**
   - `feature/功能名称` - 新功能开发
   - `bugfix/问题描述` - Bug 修复
   - `hotfix/紧急修复` - 生产环境紧急修复

2. **提交信息格式**
   ```
   <type>(<scope>): <subject>
   
   类型 (type):
   - feat: 新功能
   - fix: Bug 修复
   - docs: 文档更新
   - style: 代码格式调整
   - refactor: 代码重构
   - test: 测试相关
   - chore: 构建/工具链更新
   ```

3. **Code Review 流程**
   - 创建 Pull Request
   - 至少 1 人 approve
   - CI/CD 检查通过
   - 合并到主分支
